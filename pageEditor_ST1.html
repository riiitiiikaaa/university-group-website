2. Global CSS & Shared Template

HTML Structure

The shared template uses a standard semantic HTML structure with <header>, <nav>, <main>, and <footer> elements to clearly separate the page regions. This supports both accessibility and search engine readability.

CSS Techniques

The global CSS uses CSS custom properties (variables) defined on :root to manage the colour palette consistently. Layout is achieved with Flexbox for the header and navigation, and CSS Grid for the footer columns and card grids. Responsive design is handled through @media queries targeting 768px and 480px breakpoints.

Design & Implementation Rationale

A blue-and-cyan palette was chosen to reflect the water and ocean theme of SDG 6. Playfair Display was selected for headings to give the site an authoritative, editorial feel, while DM Sans provides clean readability for body text. The sticky header ensures navigation is always accessible regardless of scroll position.

3. Home Page

HTML Structure

The Home Page uses <section> elements for the hero, statistics strip, mission cards, and interactive gallery. Each section has a descriptive aria-labelledby attribute linking to its heading for screen reader accessibility. Cards use <article> tags as they represent self-contained pieces of content.

CSS Techniques

Page-specific embedded CSS handles the hero gradient background, the wave clip-path divider, the cards grid layout using CSS Grid with auto-fit and minmax, and the interactive gallery tiles with absolute-positioned overlays and smooth CSS transitions. A minimum of five CSS rules are defined in the embedded <style> block, all clearly commented.

JavaScript

JavaScript event listeners (mouseover, mouseout, focus, blur) are used to toggle the hovered class on gallery tiles. This drives the visual hover effect (CSS handles the appearance, JS controls the logic), satisfying the coursework requirement that interaction logic is implemented in JavaScript.

4. Gallery Page

HTML Structure

The gallery uses a grid of <div> elements with role="listitem" and tabindex="0" to make them keyboard accessible. A hidden modal dialog uses role="dialog" and aria-modal="true" to communicate its purpose to screen readers.

CSS Techniques

Page-specific embedded CSS styles the thumbnail grid, hover effects, modal overlay (using position: fixed and inset: 0), and a CSS @keyframes animation for the modal entrance. The modal is hidden by default (display: none) and shown by JS adding the modal-open class.

JavaScript

A galleryData array stores image data for all 6 thumbnails. Clicking a thumbnail calls openModal(index), which uses DOM manipulation to populate the modal with the correct image, title, and description. The close interaction is handled via a close button, clicking outside the modal, and the Escape key. User customisation (colour scheme and font style) is applied using JavaScript to the modal content area only, without affecting global styles.

6. Challenges & Lessons Learned

One of the main challenges I faced was creating the Global CSS file that all four team members would share. I had to make sure the styles were general enough to work across every page, without being so specific that they interfered with each student's own page design. I solved this by keeping the global CSS focused only on shared components — the header, nav, footer, cards, and buttons — and using clear comments to explain which parts should not be changed.

Building the gallery modal with JavaScript was another challenge. Getting the modal to close when the user clicked outside the box required checking whether the click target was the overlay element itself rather than the modal box inside it. I learned to use e.target === this inside the event listener to handle this correctly. I also learned how to move keyboard focus into the modal when it opens, which is important for accessibility.

Working collaboratively on a shared codebase taught me the importance of consistent file naming and folder structure. Early on, some navigation links were broken because teammates used slightly different file names. This taught me that agreeing on exact file names at the start of the project and sticking to them is essential for a multi-person web project.

7. Accessibility Considerations

Accessibility was considered throughout the development of all my pages. Key techniques used include:

  • Semantic HTML: Using elements like <header>, <nav>, <main>, <section>, and <footer> so that screen readers can identify page regions correctly.
  • Alt text: Every image across all my pages includes a descriptive alt attribute that conveys the image's meaning to visually impaired users.
  • Keyboard navigation: Gallery thumbnails and interactive tiles include tabindex="0" and keyboard event listeners so that users who cannot use a mouse can still interact with them.
  • Colour contrast: The primary blue (#0077b6) against white meets WCAG 2.1 AA contrast ratio requirements for normal text.
  • ARIA attributes: The modal uses role="dialog" and aria-modal="true"; nav uses aria-label; sections use aria-labelledby.
