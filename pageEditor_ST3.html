Feedback & Program Directory

HTML Structure

<section> <article> <fieldset> <legend> type="email/tel/date" <datalist> <select multiple>

The page is divided into three clearly separated <section> elements: the scrollable programme directory, the ratings & reviews summary, and the feedback form. Form fields are grouped into two <fieldset> elements ("Personal Details" and "Feedback Details") using <legend> for semantic grouping and accessibility. A <datalist> provides searchable topic suggestions, and a <select multiple> allows users to select multiple interest areas.

CSS Techniques

CSS Grid Flexbox overflow-y: auto (scroll) :hover transitions Embedded <style> CSS Variables (global)

Page-specific embedded CSS controls the scrollable programme area using max-height and overflow-y: auto. Programme cards use CSS Grid with auto-fill for a responsive layout. Form inputs use :focus states with a box-shadow ring consistent with the site's blue colour palette. All styles use the Global CSS variables for colours, ensuring visual consistency without overriding shared rules.

JavaScript Interactivity

Live character counter Custom JS validation Submission confirmation DOM manipulation Event listeners

Three JavaScript features are implemented: (1) a live character counter that updates on every input event and turns red when fewer than 50 characters remain; (2) custom form validation that checks for empty name, invalid email format (regex), missing programme selection, and short messages — displaying error messages next to each relevant field; (3) a submission confirmation that hides the form and shows a success message via DOM manipulation when all fields are valid. No data is sent to any server.

Team Page

HTML Structure

<div role="listitem"> tabindex="0" aria-label aria-hidden="true" <aside> section

Each team member is a <div> with tabindex="0" and role="listitem" so it participates in keyboard navigation and screen reader announcements. The expandable details panel is a sibling element hidden by CSS and revealed on hover/focus — no JavaScript is required for the interaction.

CSS Techniques

max-height transition opacity transition :hover / :focus selectors transform: translateY CSS Grid (auto-fill) :focus-visible outline

The hover-expand interaction uses the CSS sibling combinator: .member-block:hover .member-details transitions max-height from 0 to 320px and opacity from 0 to 1. Using max-height (rather than height: auto) enables smooth CSS animation. A transform: translateY(-5px) lifts the card on hover, and a blue box-shadow glow provides clear visual feedback. The :focus-visible pseudo-class ensures a keyboard focus ring appears only for keyboard users, not mouse users.

Design Rationale

The CSS-only hover-expand pattern was chosen because the Team page is informational — users should be able to scan members quickly without additional clicks. It also degrades gracefully without JavaScript. The design deliberately uses the same wave-bottom banner as other pages to reinforce the shared visual identity established by Student 1.

Individual Content Page — Sanitation & Hygiene

HTML Structure

<article> <aside> + <nav> href="#section-id" anchors scroll-margin-top <section id="...">

The page uses a two-column CSS Grid layout: a sticky <aside> containing an <nav aria-label="Content page sections"> with internal anchor links, and an <article> divided into six sections each with a unique id. Internal links use href="#section-id" and scroll-margin-top: 90px to account for the sticky header. All six section IDs are matched by corresponding sidebar links.

CSS Techniques

CSS Grid (2-col) position: sticky position: fixed (go-top) Image hover: scale(1.02) clamp() font sizes

The sticky sidebar uses position: sticky; top: 80px to follow the scroll without obscuring the fixed header. The "Go to Top" button is positioned with position: fixed; bottom: 2rem; right: 2rem — implemented in pure HTML/CSS as specified (no JavaScript). Article images are arranged in a responsive 2×2 grid with hover transform: scale(1.02) and a box-shadow for depth. Stat cards use a three-column grid consistent with the home page's stat-strip style.

Content Originality

All content was written originally for this coursework. The page covers sanitation statistics, the human impact of poor WASH access, six student-actionable steps, and everyday conservation habits — directly addressing SDG 6 with an action-focused, non-UN-source approach as required by the brief.

Challenges & Lessons Learned

CSS Hover-Expand Animation

Animating height: auto is not possible with CSS transitions. I initially set a large fixed max-height but the animation felt slow and mechanical. Through testing I found that max-height: 320px with a cubic-bezier(0.4, 0, 0.2, 1) easing created a natural, elastic feel. Setting this value requires knowing the panel content height in advance.

Form Validation Edge Cases

The email regex /^[^\s@]+@[^\s@]+\.[^\s@]+$/ caught most invalid emails but I initially missed the case where a user submits a form after correcting an error — the old error message stayed visible. I fixed this by calling clearError() on each field's input event, so errors disappear as users type corrections.

Lesson Learned

Writing clear HTML comments explaining what each CSS class does (and why) made it much easier to review and edit later. Code comments are not just for the marker — they are for your future self at 11pm before a deadline.

Accessibility Considerations

Semantic HTML alt text (all images) aria-label / aria-current aria-live (form feedback) tabindex="0" :focus-visible outline label for/id pairing Colour contrast (WCAG AA)

All images include descriptive alt text written to describe the visual content meaningfully — not just "image". Every form input is paired with an explicit <label for="..."> using matching id attributes. The live character counter uses aria-live="polite" so screen readers announce updates without interrupting. The form success message uses aria-live="assertive" for immediate announcement.

Interactive team member blocks use tabindex="0" and :focus-visible to ensure keyboard users can access the hover-expand panels. The active navigation link uses aria-current="page" for screen reader page context.

The site uses the primary text colour #1a2e44 on white (#ffffff), achieving a contrast ratio of approximately 13:1 — well above the WCAG AA minimum of 4.5:1 for normal text.
