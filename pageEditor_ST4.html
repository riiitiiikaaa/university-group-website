2. User Profile Logic (JavaScript)

Technical Implementation

As per Requirement 4.8, I developed a non-traditional user onboarding experience. Instead of standard forms, I utilized the prompt() method to collect user details across three distinct categories: Identity, Water Interests, and Conservation Goals.

Logic and DOM Manipulation

Data is stored in a multi-level JavaScript Object. I implemented a progressive reveal system using document.getElementById to update the profile summary in real-time. The Progress Bar is calculated mathematically by checking the length of the answered data array against the total expected inputs.

3. Sitemap Architecture (SVG)

SVG Structure

Per Requirement 4.9, I hand-coded a Scalable Vector Graphics (SVG) sitemap. The site hierarchy is depicted using <rect> for nodes and <line> elements for the branches. Using the viewBox attribute ensures the diagram is fully responsive.

Interactivity

I utilized the <a> element within the SVG to allow each node to act as a clickable link. I also applied CSS transitions on the fill and stroke properties to provide immediate visual feedback when the user hovers over a specific page in the hierarchy.

4. Water Scarcity & Conservation Page

Overview & Structure

My content page focuses on the critical issues of water stress and individual conservation. It is structured into four sections using semantic <section> tags with unique IDs to facilitate the Sticky Sidebar Navigation.

Design Rationale

To satisfy Requirement 4.10, I included four high-quality images with descriptive alt text. I used CSS Flexbox to ensure that the content remains readable across mobile and desktop devices. A persistent "Go to Top" button was added using fixed positioning for improved user experience.

5. Challenges & Lessons Learned

The most significant challenge was ensuring the SVG Sitemap's responsiveness. Initially, the lines connecting the pages would break on smaller screens. I learned to remove fixed pixel dimensions and instead use a percentage-based width combined with a responsive viewBox, which allowed the diagram to scale while maintaining its internal coordinate system.

Another challenge was handling the JavaScript Prompt workflow. If a user clicked "Cancel" on a prompt, the variable would return a null value. I implemented conditional "Sanitization" logic to ensure that if a step was skipped, the profile output would display "Not Provided" rather than the literal word "null".

6. Accessibility Considerations

  • Keyboard Focus: I ensured that every node in the SVG sitemap is focusable using the Tab key by adding tabindex="0".
  • ARIA Integration: Used aria-valuenow and aria-valuemin on the profile progress bar to assist screen reader users in understanding their completion status.
  • Color Contrast: All text and button backgrounds meet a minimum ratio of 4.5:1, ensuring readability for users with low vision.

7. JANET Compliance

I have strictly followed the JANET regulations for web publishing. All content is strictly educational and focused on SDG 6. I have avoided any unauthorized external scripts or advertising. All profile data collected from the user is stored locally in the browser's session memory only and is never transmitted to an external server, ensuring user privacy and data integrity.
