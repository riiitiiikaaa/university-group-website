4. Reflection on Validation

When I first validated my HTML pages, I encountered several common errors. Some of the main issues included missing alt attributes on images, a few unclosed tags, and instances of incorrect nesting. I also noticed minor syntax mistakes like missing quotation marks around attribute values and duplicate IDs.

To fix these errors, I carefully reviewed each validation message and updated my code accordingly. For example, I added descriptive alt text to all images to improve accessibility. I ensured that all tags were properly opened and closed, especially in sections where I had complex layouts. For nesting issues, I reorganized the structure so that elements followed proper HTML hierarchy rules. I also corrected attribute formatting and made sure each ID was unique across the page. After making these changes, I re-ran the validator and confirmed that the errors were resolved.

Through this process, I learned that writing clean and semantic HTML is very important not only for validation but also for accessibility and maintainability. Validation helped me identify small mistakes that could cause bigger issues in browsers. It also reinforced the importance of attention to detail and following proper coding standards from the beginning.
